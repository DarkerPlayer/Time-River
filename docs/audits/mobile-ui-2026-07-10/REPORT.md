# Time River 手机端 UI 审计报告

审计日期：2026-07-10  
审计范围：主计划页、移动编辑面板、时长合并、快捷完成、摘要、火漆封印、历史长河、产品说明  
验证尺寸：320×844、390×844、430×844、844×390，以及 900px 桌面切换至 390px 手机宽度  

## 总体结论

核心手机流程已恢复稳定。已输入内容会在时间格内显示；时长下拉只调整展示跨度，不再误开编辑面板；快捷完成保留在时间格右侧；有内容的窄屏时间格会自动增高，避免文字、勾选和时长按钮互相遮挡。底部菜单不固定，只有滚动到计划末尾才出现，首屏继续以时间信息为主。

## 流程检查

1. **进入计划页：健康。** 390px 首屏只显示日期切换、当天信息和时间格，没有固定菜单遮挡；页面无横向滚动。
2. **编辑时间格：健康。** 点击时间格打开独立编辑面板，输入后可明确保存或取消；保存内容立即回显并完成同步。
3. **快捷完成：健康。** 有内容的时间格右侧保留独立勾选按钮，点击后只切换完成状态，不会打开编辑面板。
4. **合并与还原：健康。** 时长下拉可从 1 小时切换到 2–6 小时，时间范围按实际占用小时显示，例如 06:00–08:00；切回 1 小时后恢复单格展示，原始事项数据不变。
5. **双日切换：健康。** 第一天和第二天在手机端分别显示，各自内容、完成状态和时长独立保存；横屏和桌面端仍可并排查看。
6. **窄屏布局：健康。** 320px 下有内容的时间格自动换为上下两行，描述与右侧控件不重叠；空白格仍保持紧凑高度。
7. **底部操作区：健康。** 同步状态、历史长河、火漆封印、摘要、打印、产品说明、封疆大吏和清空仅在页面末尾出现；清空独占整行，避免孤立半格。
8. **摘要与封印：健康。** 摘要可统计事项数、完成数和总时长；火漆封印可输入事件名称并生成只读历史版本。
9. **历史长河：健康。** 历史列表和详情无横向溢出，顶部菜单不固定；长页面滚动后仍可切换第一天和第二天，打印样式会隐藏交互控件。
10. **产品说明：健康。** 主页面底部入口可直接打开项目内说明页；390px 下目录、正文和返回入口均正常，无横向滚动。

## 已解决问题

- 移除“仅在首次加载为手机宽度时才创建内容预览”的依赖，解决旋转屏幕或缩放后已输入内容消失的问题。
- 将编辑、快捷完成和时长调整拆为三个语义明确的控件，解决选择时长时误进入文本编辑的问题。
- 修正跨小时显示的结束时间，三小时事项由 06:00 正确显示为 06:00–08:00。
- 有内容的手机时间格改为弹性高度，320px 下描述、完成按钮和时长按钮不再重叠。
- 底部菜单改为正常文档流中的两列布局，取消固定和横向滚动；清空按钮使用整行。
- 历史页顶部操作区改为正常文档流，标题使用文本节点生成，降低内容破坏页面结构的风险。
- 补充对话框、按钮、切换状态和输入控件的可访问名称及状态属性。

## 证据限制

本次已通过浏览器 DOM、真实点击流程、页面尺寸和截图检查可见布局，并验证主要控件的可访问名称、按钮状态及对话框语义。截图不能证明完整的无障碍合规性；尚未使用真实屏幕阅读器、外接键盘和不同手机系统字体放大设置进行人工测试。部署后的网络延迟、Render 冷启动和具体手机浏览器差异仍需在重新部署后做一次线上冒烟检查。
